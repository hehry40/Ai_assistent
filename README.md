## 🔍 Обзор проекта

**ИИ-администратор для Telegram** — это интеллектуальный бот, созданный на базе платформы [Suvvy.ai](https://suvvy.ai/) и языковой модели **Qwen** от Alibaba Cloud. Бот предназначен для автоматизации коммуникации с клиентами в мессенджере Telegram, предоставляя функции:

- 🗣️ Осмысленного ведения диалога с помощью нейросети
- 📚 Ответов на вопросы по корпоративной базе знаний
- 📅 Записи клиентов на встречи с синхронизацией в Google Календарь

Проект реализует концепцию «виртуального сотрудника», способного работать 24/7, обрабатывать до 100% входящих запросов и обеспечивать точность ответов до 95% благодаря технологии **Smart Bot Network** [[Suvvy]].

## ⚙️ Функциональные возможности

### 1. Осмысленное ведение диалога
```yaml
Диалоговый движок:
  - Контекстная память: до 10 сообщений истории
  - Распознавание намерений (Intent Recognition)
  - Адаптивный стиль общения (формальный/дружелюбный)
  - Обработка многозначных запросов через уточняющие вопросы
  - Поддержка вложений: текст, изображения, голосовые сообщения
```

> Бот адаптируется к любому сценарию диалога: только 1 из 600 пользователей понимает, что общается с ИИ [[Suvvy]].

### 2. Работа с базой знаний (RAG)
```yaml
Система знаний:
  - Источник: докуменды, FAQ, прайс-листы, политики компании
  - Механизм: векторный поиск + BM25-ранжирование [[10]]
  - Обновление: ручное или через Google Sheets интеграцию
  - Контроль: фильтрация «галлюцинаций» через Multi-agency архитектуру
```

### 3. Запись на встречу + Google Calendar
```yaml
Сценарий бронирования:
  1. Сбор данных: имя, контакт, услуга, предпочтительное время
  2. Проверка доступности: запрос к Google Calendar API
  3. Согласование: предложение слотов клиенту
  4. Подтверждение: создание события + отправка уведомления
  5. Синхронизация: обновление в календаре и базе клиентов
```

> Интеграция с Google Calendar позволяет боту проверять доступность, предлагать слоты и автоматически вносить встречи в расписание [[Suvvy]].

## 🎬 Сценарии использования

### Сценарий 1: Консультация по услугам
```
Клиент: "Какие услуги вы предоставляете?"
↓
Бот: 
  1. Запрос к базе знаний (RAG)
  2. Формирование структурированного ответа
  3. Предложение записаться на консультацию
```

### Сценарий 2: Запись на встречу
```
Клиент: "Хочу записаться на завтра в 15:00"
↓
Бот:
  1. Проверка доступности в Google Calendar
  2. Если слот занят → предложение альтернатив
  3. Если свободно → подтверждение и создание события
  4. Отправка уведомления клиенту и менеджеру
```

### Сценарий 3: Обработка возражений
```
Клиент: "А почему так дорого?"
↓
Бот:
  1. Распознавание намерения "работа с возражением"
  2. Поиск аргументов в базе знаний (ценность, гарантии, сравнение)
  3. Персонализированный ответ в тоне бренда
  4. Предложение бонуса / демо-сессии для конверсии
```

### Сценарий 4: Эскалация к человеку
```
Триггеры:
  - Клиент запросил оператора
  - Низкая уверенность ответа (< 70%)
  - Сложный юридический/технический вопрос
↓
Действия:
  - Уведомление менеджера в Telegram
  - Передача контекста диалога
  - Плавный переход к живому оператору
```

## ⚙️ Настройка и развёртывание

### Этап 1: Подготовка (5-15 минут)
1. [ ] Регистрация в [Suvvy.ai](https://suvvy.ai/) и получение бонуса $5
2. [ ] Создание бота в @BotFather, получение токена
3. [ ] Настройка проекта в Suvvy: выбор шаблона «AI-Administrator»

### Этап 2: Настройка ИИ-ядра
```yaml
# Системный промпт для Qwen (пример)
system_prompt: |
  Ты — виртуальный администратор компании [Название].
  Тон общения: дружелюбный, профессиональный, лаконичный.
  Правила:
  - Всегда проверяй базу знаний перед ответом
  - Не придумывай цены и условия — если нет данных, предложи связаться с менеджером
  - При записи уточняй: имя, телефон, услугу, удобное время
  - Если клиент нервничает — проявляй эмпатию и предлагай помощь человека
```

### Этап 3: Наполнение базы знаний
- Загрузка документов: PDF, DOCX, TXT, ссылки
- Структурирование через тегирование: `#цена`, `#услуги`, `#политика`
- Тестирование поиска: проверка релевантности ответов

### Этап 4: Интеграция с календарём
1. Включение интеграции Google Calendar в Suvvy
2. Авторизация через OAuth 2.0
3. Настройка правил бронирования:
   - Длительность встречи по умолчанию
   - Буфер между встречами
   - Рабочие часы и выходные

### Этап 5: Тестирование и запуск
```bash
# Чек-лист перед запуском
✅ Тестовые диалоги: 20+ сценариев
✅ Проверка RAG: точность ответов > 90%
✅ Интеграция: создание события в календаре
✅ Эскалация: корректная передача оператору
✅ Логи: запись всех диалогов для аналитики
```

## 📎 Приложения

### Приложение А: Пример системного промпта
```markdown
# РОЛЬ
Ты — ИИ-администратор компании «[Название]». Твоя задача — помогать клиентам:
- Отвечать на вопросы об услугах, ценах, условиях
- Записывать на консультации и встречи
- Передавать сложные запросы живому оператору

# СТИЛЬ
- Тон: дружелюбный, профессиональный, без излишней формальности
- Длина ответа: 2-4 предложения, если не требуется детализация
- Эмодзи: умеренно, только для улучшения читаемости ✅

# ПРАВИЛА
1. ВСЕГДА сверяйся с базой знаний перед ответом
2. НЕ придумывай цены, акции, условия — если данных нет, скажи: 
   «Уточню у менеджера и вернусь с ответом в течение 15 минут»
3. При записи ОБЯЗАТЕЛЬНО уточни: имя, телефон, услугу, удобное время
4. Если клиент выражает недовольство — прояви эмпатию и предложи помощь человека
5. Завершай диалог вопросом: «Чем ещё могу помочь?»

# ФОРМАТ ОТВЕТА ДЛЯ ЗАПИСИ
Если клиент хочет записаться, отвечай строго по шаблону:
«Отлично, [Имя]! Для записи на [услуга] уточните, пожалуйста:
• Удобную дату и время
• Контактный телефон
• Предпочтительный способ связи (звонок / сообщение)»
```
